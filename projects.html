MobContracts

MobContracts was a Minecraft plugin that I spent countless hours working on. This was a self-assigned project to practice using Java and other technologies. It eventually spiraled into become a full-time project in which there was a final goal - creating the best boss-based Mob plugin out there! It consisted of custom-created GUIs, multiple databases and even integration of other plugins to work with them. A couple of key features I created was custom messages, prefixes, events and effects. It also included a database to go alongside the "profile" portion of the plugin in which players could view their own, as well as other player's progression through the bosses. It also included a fully customisable configuration file in which server admins could add their own unique twist to things. Changing everything they possible could from colours, to full move-sets. This was a project I was most proud of.

Periphery

Perhiphery was a group-based university project in which students were tasked to create a game using a special technology some of which included VR headsets and special controllers. We decided to use an eye-tracker called Tobii. We utilised this to create a game where you have to ensure you're always looking away from the monster, otherwise it would start tracking you. We preyed on the natural human urge to look at anything that's moving, especially in people's peripherals. The game included two monsters. One slow, one fast. The player had to complete puzzles to complete the game, without the monster catching them. The monster also had noise and proximity checks. I was tasked with all of the monster's AI. Ensuring it worked smoothly and effectively. We were voted best game concept for multiple weeks in a row, but sadly didn't make it to the games convention.

MetroCore

MetroCore was another plugin written for Minecraft, however, this one was server specific. I was approached by a friend of mine who had been supporting me through my Java journey asking to create a Minecraft plugin for his server that would be set to replace the ever popular "Essentials" plugin. The plugin included commands such as /admin and /mod to enable or disabled mod or admin mode which allowed different permissions, as well as, a full reporting and suggestion system all linked to multiple databases that worked asynchronously to ensure no stuttering on the main thread. It also included discord integration that would allow players in-game to talk on Discord too, and visa versa.

Project: Main Menu

This project was also part of a University assignment that was set in which we had to demonstrate our use of visual programming and C++ in Unreal Engine to create a visually appealing, but function main-menu system. I chose to go for a dark, moody main menu that could be used on a horror game or a game that used a lot of suspense. It utilised a video for the background which was raining, as well as the use of blurring to focus. It also included an achievements tab that included tooltips.

My Portfolio

To me, my portfolio is where I can express myself. I can use my favourite colours, favourite fonts and fully customise it exactly how I want it to be. I have full creative freedom in my portfolio. However, like any other project I do, I set myself a challenge. I wanted my portfolio to view non-scrollable on desktop. I didn't want there to be a single page in which you're forced to scroll to view. I wanted it all there, ready, for the user immediately.

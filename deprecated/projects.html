MobContracts

MobContracts was a Minecraft plugin that I spent countless hours working on. This was a self-assigned project to practice using Java and other technologies. It eventually spiraled into becoming a full-time project in which there was a final goal - creating the best boss-based Mob plugin out there! It consisted of custom-created GUIs, multiple databases and even integration of other plugins to work with them. A couple of key features I created was custom messages, prefixes, events and effects. It also included a database to go alongside the "profile" portion of the plugin in which players could view their own, as well as other player's progression through the bosses. It also included a fully customisable configuration file where server admins could add their own unique twist to things. Changing everything they possible could from colours, to full move-sets. This was a project I was most proud of.

Periphery

Perhiphery was a group-based university project in which students were tasked to create a game using a special technology some of which included VR headsets and special controllers. We decided to use an eye-tracker called Tobii. We utilised this to create a game where you have to ensure you're always looking away from the monster, otherwise it would start tracking you. We preyed on the natural human urge to look at anything that's moving, especially in people's peripherals. The game included two monsters. One slow, one fast. The player had to complete puzzles to complete the game, without the monster catching them. The monster also had noise and proximity checks. I was tasked with all of the monster's AI. Ensuring it worked smoothly and effectively. We were voted best game concept for multiple weeks in a row, but sadly didn't make it to the games convention.

MetroCore

MetroCore was another plugin written for Minecraft, however, this one was server specific. I was approached by a friend of mine who had been supporting me through my Java journey asking to create a Minecraft plugin for his server that would be set to replace the ever popular "Essentials" plugin. The plugin included commands such as /admin and /mod to enable or disabled mod or admin mode which allowed different permissions, as well as, a full reporting and suggestion system all linked to multiple databases that worked asynchronously to ensure it worked fluidly alongside other plugins. It also included discord integration that would allow players in-game to talk on Discord too, and visa versa.

Project: Main Menu

This project was also part of a University assignment that was set in which we had to demonstrate our use of visual programming and C++ in Unreal Engine to create a visually appealing, but function main-menu system. I chose to go for a dark, moody main menu that could be used on a horror game or a game that used a lot of suspense. It utilised a video for the background which was raining, as well as the use of blurring to focus. It also included an achievements tab that included tooltips.

Asteroids

PlayBuffer is an open source C++ library for making simple two dimensional games inside Windows console. Basically, it's a game engine to run in command line. It was brought to our attention by my University tutor as one of the options for our 2D game assignment. As someone who had barely worked in C++ prior to this, I found this both challenging but interesting. I found it an interesting and informative way of teaching myself the basics of C++, whilst also completing the assignment set. I created asteroids, a game I thoroughly enjoyed playing when I was younger.

Cambot

Cambot is my second Discord bot project. It first started as a Discord bot with nothing but random API calls, such as Dad Jokes and Dog images. However, it quickly became a project I was passionate about. This is the third iteration of Cambot, and is currently still under development. The new bot also included mini-games, educational commands, a full levelling system and much more! The bot was also built to be run on a RasperryPi. Plenty use of optimisation techniques, alongside efficient programming allows there to be no issue running on a computer so small. Cambot is currently in the midst of being worked on, and optimised further, and will hopefully be back live soon!

Custom Maths Library

This project was a University project, however, it's something that I thoroughly enjoyed. We were set the task of creating a custom maths library in which we can use to work alongside the Unity physics engine. Some examples would be custom vectors, all the way up to 4 dimensional, as well as, all of the functions that co-inside with Vectors and other. The main part of the assignment was using it for a tower defence style game where the projectiles would use the custom library. However, this portfolio piece is more to outline the Maths library.

AI Population simulation

A University assignment where we were tasked to use two seperate types of AI to create a predator VS prey style simulation. The predator should hunt the prey, and the prey should be run away from the predator. Prey were also needed to "breed" to pass down genes gained throughout the previous generations life. I found this particularly hard due to my lack of AI knowledge, however, I found it a good starting point to learning more about the ins and outs of AI creation and training.
